# Общий обзор

SkillBridge создаётся как pet-проект уровня коммерческого веб-приложения.

Основная цель выбранного стека — показать навыки современной backend-разработки на Django с использованием лучших практик индустрии.

Проект намеренно использует технологии, которые наиболее часто встречаются в вакансиях Middle / Middle+ Backend Developer.

# Backend

## Python 3.13

### Почему Python

Python остаётся одним из наиболее популярных языков разработки серверных приложений.

Он обладает:

- высокой читаемостью;
- огромной экосистемой библиотек;
- быстрым временем разработки;
- большим сообществом;
- отличной поддержкой Django.

Для проекта SkillBridge Python позволяет сосредоточиться на архитектуре приложения, а не на низкоуровневых деталях реализации.

---

## Почему именно Python 3.13

Используется последняя стабильная версия языка.

Преимущества:

- улучшенная производительность;
- оптимизация интерпретатора;
- новые возможности typing;
- более быстрый запуск приложений;
- поддержка современных библиотек.

Использование актуальной версии демонстрирует применение современных инструментов разработки.

---

# Django 5.x

Django является основным backend-фреймворком проекта.

Он отвечает за:

- маршрутизацию;
- ORM;
- административную панель;
- систему авторизации;
- обработку запросов;
- шаблонизатор;
- работу с формами;
- middleware;
- безопасность.

---

## Почему Django

Причины выбора:

- зрелый фреймворк;
- огромное количество готовых решений;
- встроенная ORM;
- качественная система безопасности;
- Admin Panel;
- удобная миграционная система;
- высокая скорость разработки.

Для pet-проекта уровня production Django является практически идеальным выбором.

# Database

## PostgreSQL

Основная база данных проекта.

Используется для хранения:

- пользователей;
- профилей;
- навыков;
- заявок;
- отзывов;
- уведомлений;
- связей между моделями.

---

## Почему PostgreSQL

Причины выбора:

- одна из лучших СУБД для Django;
- высокая производительность;
- полноценная ACID-транзакционность;
- отличная поддержка индексов;
- полнотекстовый поиск;
- JSONB;
- расширяемость;
- широкое применение в коммерческой разработке.

---

## Используемые возможности PostgreSQL

Планируется использование:

- Foreign Keys
- Composite Indexes
- Partial Indexes
- Unique Constraints
- Check Constraints
- Transactions
- Full Text Search
- EXPLAIN ANALYZE при оптимизации запросов

---

# Redis

Redis используется как in-memory хранилище.

На первом этапе проекта:

- кэширование;
- хранение временных данных.

На следующих этапах:

- Celery Broker;
- Celery Backend;
- очереди задач;
- кеширование запросов;
- rate limiting.

---

## Почему Redis

Redis обладает:

- очень высокой скоростью;
- поддержкой TTL;
- структурами данных;
- простотой интеграции с Django.

# Infrastructure

## Docker

Каждый компонент приложения запускается внутри собственного контейнера.

Преимущества:

- одинаковое окружение;
- простое развёртывание;
- отсутствие проблемы "работает только у меня";
- масштабируемость.

---

## Docker Compose

Используется для локальной разработки.

Сервисы:

- Django
- PostgreSQL
- Redis
- Nginx

---

## Gunicorn

Gunicorn является WSGI-сервером приложения.

Используется между Django и Nginx.

---

## Nginx

Nginx выполняет функции:

- Reverse Proxy;
- раздача статических файлов;
- раздача медиа;
- SSL termination;
- балансировка нагрузки (в будущем).

# Code Quality

Для обеспечения единого стиля используются следующие инструменты.

## Black

Автоматическое форматирование кода.

---

## Ruff

Проверка:

- ошибок;
- потенциальных багов;
- стиля;
- импортов.

---

## pre-commit

Перед каждым коммитом автоматически запускаются:

- Black;
- Ruff;
- дополнительные проверки.

Это позволяет предотвратить попадание некорректного кода в репозиторий.

# Почему выбран именно этот стек

При выборе технологий учитывались следующие критерии:

- востребованность на рынке;
- совместимость компонентов;
- зрелость технологий;
- производительность;
- безопасность;
- простота сопровождения;
- возможность масштабирования;
- соответствие современной коммерческой разработке.

Все технологии широко применяются в production-системах и хорошо интегрируются друг с другом.
